三条设计原则

1 · 预览是房间的窗户 NEW

不再是门内证据:任务任何状态(进行中 / 待审查 / Full 模式 / 已接受)右栏都可用。 agent 写一笔、HMR 刷一笔,你实时看着页面长出来——这是 Full 模式用户唯一的观察窗, 也是本次两个实测坑(找不到入口、Full 跳过门)的结构性解法。

2 · 反馈落回同一个 composer NEWLovable/Windsurf

选中元素(S)或圈选区域(R)后不弹便签窗——chip(截图缩略 + selector + 尺寸)落进 Room composer,你在熟悉的输入框里说话,回车即回炉。一个输入口,一条对话, 附件语义与 @ 文件一致;门内旧便签流程随之退役。

3 · 检测只点亮,展开靠意图 KEEP

端口探测到只在标题栏亮一枚徽章(外部 CLI 检测的教训:布局永不因弱信号自动动); 点徽章右栏滑出,再点收回,宽度可拖,记住偏好。空态一键起服务(am.1)原样保留。

不变的部分与刻意不搬的

边界原样 KEEP

端口归属=任务自己的树(cwd 判定);iframe 沙箱 + loopback CSP;门内三 tab 保留, Preview tab 变成右栏的评审投影(同组件同探测);PR 草稿、GIT-007、 superseded/stale/Unverified 语义一字不改。

元素拾取的实现注记 NEW · 需 ADR

跨源 iframe 拿不到 DOM——方案:主进程 webFrameMain.executeJavaScript 只对loopback 任务树页面注入拾取器(hover 高亮,点选 postMessage 回 {selector, rect, text}),失败或用户拒绝时自动退回圈选(零注入,任何页面可用)。 注入范围与开关写进 ADR。

console 自愈的护栏 NEW

捕获零注入(主进程 console-message 按来源端口过滤); 自动转发仅限 error 级且落在 agent 最近一次写入的时间窗内(它弄坏的它自己修); 同一错误去重、每次写入至多回传一批、每 run 限频;warning 只计数; 设置项 preview.consoleToAgent(auto / manual / off)。 页面里第三方脚本的既有报错不在时间窗内,不会冤枉 agent。

不搬 SKIP

v0 属性面板(margin/color 可视化调参)——表达一律走语言;Lovable 行内改文字(T)—— 需注入回写文件,归属语义复杂,V1 不做;Replit agent 自测截图——已有 VER 轴,不混; 云端分享预览 URL——仍是独立 ADR。
