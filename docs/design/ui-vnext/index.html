让人「卧槽」的五个方向

产品已经有四套 skin（Studio / Archive / Terminal / Index），所以这次大升级不能只是第五套配色—— 每个方向都在 结构、材质、活性、叙事 里押注一种「第一眼震撼」。五个 mock 全部使用真实的产品概念 （Session Rail、Needs You、三段门、验证、worktree 隔离、回放），数据是同一个虚构工作日，方便横向对比。

判断框架：「卧槽」从哪来？A 赌的是 活性（一屋子智能体在干活的画面）；B 赌 反潮流的工艺（全行业深色赛博，我们做精装刊物）； C 赌 材质（最上镜的玻璃与光）；D 赌 独有能力（竞品没有全程记录，做不出可拖动的时间）；E 赌 规模化效率（经营车队的操作系统感）。 它们并不互斥——见页尾的组合建议。
C+Fusion

组合方案 · Obsidian 底座三场景二稿 · 2026-07-19

黑曜石玻璃气质 × 三个场景职责分明：Home = 主页就是 Chat 输入（默认打开，保持现状， 只加一行可点的态势摘要）；Deck = 纯监控新入口（◎ 第五 tab，无任何输入框，右上 Cards ⇄ Rows 收纳 E 的密度模式）； Room 底部 = D 的胶片缩略版（细带常驻、hover 升起、可拖动回看）。现有四入口零改动。

回答：「主页不动、不塞爆现有 tab 的前提下，把三个卧槽点装进一个壳」 同构对照版 → fusion-atelier（换纸感语言） 打开组合 mock →
B+Fusion

组合方案 · Atelier 底座与上面完全同构，只换气质 · 2026-07-19 追加

印刷工作室气质的同一套结构：Deck 是「今日版面」（数字栏 + 头条印章 + 专栏）， Rows 密度转译为「印刷账本」行情表，Room 底部的胶片转译为「装订线」（红棕缝针播放头）。 两个组合版并排看，就是纯粹的 B ⇄ C 气质二选一。

回答：「同一套 IA 结构穿上纸与活字，还成立吗？」 同构对照版 → fusion-obsidian（玻璃语言） 打开组合 mock →
组合版的 IA 决策（回应「要不要新增 Tab」+「主页被改了？」）：主页 = Chat 输入，保持现状且是默认页； 态势/密度看板是「监控+就地决策」，与 Sessions 导航、与主页发起都是不同节奏——所以新增第五个 activity 入口 Deck 专门承载它，Deck 里没有任何输入框（发起回主页），现有四入口与面板零改动；E 的行看板不占 tab，是 Deck 内的密度切换。 附带观察：Deck 已含 Needs You 通道 + 徽章，中期可评估 Inbox 入口是否可省（有机会减回四个）。
SSkins

Skin 适配演示 · 同一套 Deck，七种皮肤「适配还是新增」的答案 · 2026-07-19 追加

同一份 Deck 组件代码（只写一遍，全语义 token + 三个新效果 token），在 四个现有 skin（自动获得安静版新结构）与 两个第五-skin 提案（Obsidian / Atelier，点亮玻璃与辉光） 之间实时切换。呼吸 / 心跳 / 扫光在每种皮肤下都保留（是动画+语义色，不是材质），只有玻璃模糊与辉光是新 skin 专属。

回答：「新增一个样式，还是在四个样式上适配？」——答案是分层：结构全适配，材质归新 skin。 实施 = 组件 token 化一遍 + theme.css 多一列值，不是五套代码 打开演示（点皮肤切换）→

给同时经营 10+ 会话的重度用户：一切都是行——状态灯、实时动作列、三段门微条、等宽 diffstat 严格对齐； j/k 航行、行尾单键操作（a approve / g grant / r review）、多选浮出批量决策条、⌘K 直达任何门。

怎么选：一道正餐，两道配菜

立场组合（推荐思路）：先选 气质立场（B 的纸与活字 vs C 的黑与光——二选一，这是品牌级决定）， 再叠 结构升级：A 的态势带/发光通道可以直接进 Launcher，D 的时间胶片可以收进 Room 底部（默认折叠、hover 展开）， E 的行看板作为 Sessions 面板的「密度模式」给重度用户切换。

最小惊艳路径：C（或 B）定材质 → A 的 Needs You 通道 + 会话心跳进 rail → D 的胶片先做只读缩略版。 三步每一步都可独立发布，每一步都有一个可传播的「卧槽点」。

工程红线（不因视觉升级而破）：动画预算（不聚焦即冷却、reduced-motion 全停）、humane language、 无 emoji chrome、审阅门永远显式 —— 五个 mock 都已按这些约束绘制。
