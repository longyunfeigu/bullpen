Session 上下文附件 · 4 个方向

目标:在 Session 视图里能把文件、文件夹、图片作为上下文交给代理 —— 既支持拖拽,也要能看到项目里有什么。四款是本质不同的交互模型,可任选其一,也可组合(例如 A+D、C+D)。

现状问题:「拖文件喂上下文」在 Home 已实现(PIVOT-015:侧栏项目树可拖、 接 Finder 拖放、落成 chips)——但你日常所在的 Session 视图是缺口:看不到任何文件树、 不接 Finder 拖放、@ 只会插行内 @path 纯文本,也没有图片附件。 想把 public/index.html、一张设计稿截图或整个 styles/ 目录交给代理时,无处可拖、无处可选。 四个方向 = 四种"补齐 + 统一两套现状模式"的策略。
A

文件抽屉 — 挂在输入框上的目录树

上下文属于输入框,浏览也发生在输入框旁:📎 打开一个悬浮抽屉(树 + 搜索 + 会话改动),拖或点 ＋ 落成输入框上方的引用条。

  • 树行可拖到输入框;悬停出 ＋ 一键加入;文件夹整目录引用
  • 输入框本身是 Finder 拖放区(文件/文件夹/图片,真实可试)
  • 引用条沿用 CodeContextRef 已确认的「引用在正文之上」样式
✓ 不动布局、按需出现;浏览与投放零距离;与 @ 天然互补 △ 抽屉是暂态的 —— 想"一直看着目录"的人不满足
打开方案 A a-composer-drawer.html
B

会话内文件面板 — Sessions / Files 双页签

左侧栏本身分成 Sessions 和 Files 两个页签:文件树常驻在会话旁边,任何行都能长途拖进输入框或对话区。

  • 树常驻、带会话改动徽标(+5 −2),随时可见"项目里有什么"
  • 拖放目标有两个:输入框、整个对话区(都会亮起投放框)
  • 活动栏的文件夹图标即入口,与今早 Files 视图重构一脉相承
✓ IDE 直觉、可见性最强;树 ↔ 会话来回最顺手 △ 占掉 Sessions 列表;背离"会话视图不放第二 Explorer"的既有约束,需要正式改约束
打开方案 B b-session-explorer.html
C

全窗拖放 + Context 页签 — 上下文一等公民

不放树:文件从它们所在的地方(Finder / 截图)来。拖进窗口任意位置,浮层给两个投放目标 —— 这条消息 或 固定为项目上下文;右侧新增 Context 页签统一管理。

  • 全窗都是靶区,目标大、不用瞄准输入框
  • 首创"作用域":一次性(随消息)vs 固定(项目每个新 Session 自动带)
  • Context 页签汇总草稿引用、固定项、历史消息引用,可逐项移除
✓ 上下文全生命周期可见可管;固定上下文是差异化能力 △ 概念最重;在应用内浏览/挑选项目文件仍需别的入口(可与 D 组合)
打开方案 C c-drop-anywhere.html
D

@ 内联引用 — 引用长在句子里

零新面板:输入 @ 在光标处唤起模糊搜索,引用作为 token 直接嵌进句子("对照 @设计稿.png 改 @index.html");拖拽落点同样变成句中 token。

  • @ 菜单分组:建议(会话改动)/ 文件 / 文件夹 / 图片,键盘全程可达
  • 引用在语义位置上 —— 代理确切知道每个引用修饰哪句话
  • 发送后 token 原样进时间线(见 mock 中已发送的示例消息)
✓ 最快的键盘路径、零 UI 成本;心智与 Cursor/Claude Code 一致 △ 只能搜不能浏览;引用多了句子会变长(适合与 A 或 C 组合)
打开方案 D d-inline-mentions.html
怎么看这些 mock:每页顶部深色条是 mock 场景控制器(非产品 UI),含「演示 / 重置」;四页都接了真实拖放 —— 直接从 Finder 拖文件、文件夹或图片进页面即可看到落点行为。
与现有实现的关系:A ≈ 升级现有 @ 文件拾取器(加树/拖拽/文件夹/图片);B ≈ 把 Home 已有的可拖项目树(HomeProjectTree)搬进会话视图常驻;C ≈ 把 Home 已有的 OS 拖放补进 Room + 新增「固定为项目上下文」作用域(新概念,需定义持久化与权限边界);D ≈ 把 Room 现有的行内 @path 纯文本升级为原子富 token。内部拖拽 MIME 与产品一致 (application/x-charter-ref)。
与既有决策的关系:引用条样式沿用 CodeContextRef mock 已确认的「结构化引用位于正文之上」;方案 B 触碰了 unified mock 里"会话视图不引入第二 Explorer"的约束,若选 B 需在 ADR 里正式修订。无论选谁,都要回答:Home 的 chips 与 Room 的行内引用统一成哪一种?四款均避免出现引擎品牌字样。
