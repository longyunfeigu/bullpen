Mockup：History 收录「在 Charter 里开的、已退出」的 Claude/Codex 会话。
1会话退出且不需要你再做决定（没改文件，或改动已接受/回滚）→ 自动挪进 History，不再堆在项目组里。
2退出但改动还没审的，仍留在项目组并亮 Review 徽章——History 永远不藏需要决定的东西（沿用现有原则）。
3History 行 hover 出现 ↻ Resume：点击开新终端接着原对话继续聊（产品记着每个会话的精确 ID，`claude --resume` / `codex resume` 直达）；旁边是已有的归档按钮。
4点行本身照旧打开该会话的房间回看；归档语义不变（archive 后彻底收起）。
5只收产品内启动的会话；你在外面终端跑的不扫、不进来。
行样式完全沿用现有 rail：品牌标（✳ Claude 橙 / ▣ Codex / ◆ Charter）、灰点=已结束、绿点=活着、「Ended」灰徽章。History 里的行会多显示项目名前缀（已有行为，右侧第二行灰字）。
