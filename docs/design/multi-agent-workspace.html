PRODUCT NOTE · DRAFT 03

让模型并行工作，而不是让工作区围着模型转

真正自然的工作台，不应该要求你先做一个影响全局的选择：选 Claude，就失去 Codex；切到另一个目录，就把刚才打开的上下文全部带走。

编辑器负责承载你正在看的材料，会话负责声明 Agent 能够操作的边界。 两者可以互相关联，但不应该绑定成同一个状态。

因此，每个 Agent 都有独立的 working set、cwd、权限和任务历史。Claude 可以在发布稿里润色句子，Codex 同时在另一个仓库里跑测试。你看到哪，跟随哪，但其他任务不会被暂停。
