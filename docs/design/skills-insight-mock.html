Charter — Skills 透视:触发统计 + context 预算
三个方案共享同一份状态——在任何一处把 skill 切到 Off,预算条、总计、候选清单全部实时更新。 数据来源:调用次数 = turn ledger 里 45 天内 load_skill 的聚合;token 占用 = skill-store 为每个 enabled skill 生成的 <skill> preamble 块长度估算。 Off 的 skill 不进 preamble,所以 token 记 0,但历史调用次数保留(帮你判断"要不要再开回来")。
方案 A · 最小改动 —— 留在 Settings → Agent 卡片内

Skills

每行右侧加两列:45 天调用(次数 + 周粒度 spark 柱,末次调用时间)与 token 占用(数字 + 相对占比条)。卡片顶部一条预算摘要:enabled skills 合计占多少 token、约占 200k 窗口百分比。零调用且 enabled 的行出现 unused 45d 徽章——这就是断舍离的最小闭环,不动信息架构。

方案 C · 断舍离审查 —— 从 B 页 “Review usage…” 进入

Review skill usage

last 45 days
按“每次调用成本”(token ÷ 45 天调用次数)排出候选;零调用排最前。勾选后批量禁用—— 随时可在列表里一键开回来,不删除、不丢历史。
审查是建议不是自动化:Charter 只排序和算账,禁用永远是用户点的。explicit-only / side-effects 徽章照常显示,提醒“这个 skill 平时靠手动 / 唤起,调用少不代表没用”。
