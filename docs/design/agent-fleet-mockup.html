对照：FanBox 怎么做的，这里升级了什么

  • FanBox 只有 tab 条 + 状态圆点 + ⚡ 闪 8 秒：审计是瞬时的，闪完无痕；看舰队要挨个点 tab。
  • 本稿把 ⚡ 升级为常驻指挥链标记（⌁ 由 t1 指挥）：控制关系存在多久，标记亮多久，并写明是谁（FanBox 只知道"被摸"，不知道"谁摸的"）。
  • tab 圆点语义沿用 FanBox 的忙/闲/退出（绿呼吸/灰/红），成本低、已被验证。

为什么推荐 A（网格 + 指挥树 + 活动流）

  • 网格答"大家都在干嘛"（用户诉求：一个窗口看全部 agent 工作）；指挥树答"谁指挥谁"；活动流答"刚才发生了什么"。三问三答，各占一块。
  • 刻意不做节点连线画布：终端是需要面积的矩形，连线浪费空间且必然交叉；控制关系基数很低（1 指挥 N），树 + ⌁ chip 表达足够。连线图适合 replay 的静态回顾，不适合干活时的常驻界面。
  • 方案 B 是保守替代：改动小（现有单窗 + 右侧栏），但"一眼看全"不成立，只推荐在窄窗口时作为 A 的降级形态，而非独立方案。

审批与接管的交互契约

  • 审批卡长在目标瓦片上（哪个终端要被注入，卡就在哪），而不是全局弹窗——空间即语境；侧栏/角标提供全局计数入口。
  • 选项三档：允许一次 / 本任务内总是允许（落成 StandingRule）/ 拒绝。拒绝同样入账。
  • 点击瓦片正文即接管（试试 t3）：亮蓝色缎带，主指挥的后续注入排队暂挂；「归还控制」后放行。人永远赢过 agent。
  • 工具条「⏸ 暂停全部遥控」是总闸：一键让所有指挥链停摆，窗口保持原样。
