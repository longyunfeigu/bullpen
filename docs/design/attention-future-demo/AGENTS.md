# Prototype Instructions

Run the local server yourself and open the preview in the browser available to this environment. Do not give the user server-start instructions when you can run it.

Before making substantial visual changes, use the Product Design plugin's `get-context` skill when the visual source is unclear or no longer matches the current goal. When the user gives durable prototype-specific design feedback, preferences, or decisions, record them in `AGENTS.md`.

When implementing from a selected generated mock, treat that image as the source of truth for layout, component anatomy, density, spacing, color, typography, visible content, and hierarchy.

## Prototype-specific decisions

- This is one comparison prototype containing three genuine product directions, not three color themes.
- Each direction uses the same 35-second story: Home Chat Session launch → exact session/attention jump → sub-agent work → scoped permission → semantic terminal completion.
- After the shared Home entry, the primary comparison dimensions are the dominant surface (Home + Attention, durable Task Room, or Terminal Workbench) and where human attention re-enters the flow.
- Preserve Charter's single-right-sidebar principle and Gateway/audit boundary in every direction.
- Home's Chat Session composer is the primary product entry and must remain visible in every direction. Attention is a Home summary plus an Inbox destination, never a replacement for session creation.
