五条投影规则（大白话版）

问题不是"记多了"，是"全都端上桌了"。记录照旧全记，默认视图只按下面五条挑着显示：

  1. 1折叠条别比内容多藏 1~2 条还立一块牌子，等于占一行去省一行。少于 3 条：要么直接显示，要么写进邻居的小字里。
  2. 2排场跟着任务大小走语义节点 ≤ 6 的小会话进"紧凑模式"：没有折叠条、没有章节，就是一条能一眼读完的流水。
  3. 3"你批准了"是贴纸，不是一张卡批准是对某件事的回应，贴在那件事上（✓ 已批准 19:13），点开看审计详情。不再单独占两行。
  4. 4错误说人话，红色留给真出事的WS_DIR_UNREADABLE 这种自己恢复了的小毛病，降级成黄色一行"目录暂不可读（已恢复）"，原始错误码收进展开详情。
  5. 5徽章反着打行行都写"结构化记录"等于没写。默认不打标，只有例外才打：推导叙事 / 引自最终报告 / 观察记录。

⛃ 账本不动：权限审计、settlement、原始错误码全部照记，探究层 / 核验层照样能看到每一条。改的只是"回顾"这一层给谁看多少。

用户什么时候真会看这些记录

  • 99% 的时候：只想确认"干了什么、成了没"。看结论 + diff + 有没有黄色/红色，10 秒离开。
  • 1% 的时候（它干了不该干的 / 审计）："我批准过什么？"这时带时间戳的批准记录是黄金——所以它要记全，但不需要平时摆出来。

大任务不受影响

  • 语义节点 > 6 的会话（如 replay-v4 mock 里的 401 修复，14 节点）保持现有章节 + 折叠条形态——那个规模下折叠条是正压缩（一行藏 87 条）。
  • 规则 1/3/4/5 对大小任务统一生效；只有规则 2（紧凑模式）按规模分档。
