外部 CLI 终端嵌入 — 交互方向对比

本机已有 Codex CLI / Claude Code 等编码代理终端。目标：把它们嵌进 Charter， 并利用产品已有的能力（FS watcher、agent change 装饰、ADR-0014 的 L1 file peek live-follow、Changes rail、Review 流程）让「外部 agent 改文件」获得实时可见的 live demo 感。四个方向按集成深度递增排列；D 最浅、C 最深。 每个演示会自动循环播放，也可点「重播」。

docs/design · 2026-07 · 配套讨论：终端经 PTY（node-pty + xterm.js）嵌入 utility process；外部 CLI 不经过 Tool Gateway，是权限引擎外的「未托管」写入者 —— 各方向都要回答这一点。
方向 A

终端即房间 — 外部会话成为一等 Task Room

终端占据对话列的位置；FS watcher 把 CLI 的改动实时提升进 Changes 栏与 L1 peek
复用最大化：Changes rail、peek 分屏、live-follow、Review 流程全部原样可用；对 CLI 零解析——只看文件系统和 git，任何 CLI（今天的和未来的）都兼容。
时间线没有语义：房间里看不到 thinking/工具卡片，只有原始终端；外部写入在权限引擎外，需要「未托管」徽章 + 自动 snapshot 兜底。

可叠加的点子（不挑方向，都能加）

这些是「你可能没想到的」——大多与 A–D 正交，可作为任一方向的增强。

中成本 · 高价值

会话发现与接管

扫描 ~/.claude/projects/*.jsonl 与 codex 的 sessions 目录，把「今天在别的终端里跑过的会话」列进 Home；一键 claude --resume <id> 在房间里继续。IDE 不只是嵌终端，而是接管你散落在全机的 agent 会话。

中成本 · 高价值

双向桥：用产品 composer 驱动 CLI

房间的卡片 composer 直接把文字写进 CLI 的 stdin：@文件引用 chips 展开成路径、拖拽文件进来、模型/effort 选择映射成 CLI flag。终端负责输出，输入体验保持产品级。

低成本 · 必做

安全边界徽章 + 自动快照

外部 CLI 不经过 Tool Gateway/Permission Engine，是「未托管」写入者。房间/终端 tab 常驻 EXT 徽章；会话开始时自动 snapshot（复用现有机制），提供一键回滚——把架构红线转化为可见的产品语义。

低成本

终端内路径可点 → peek

解析终端输出里的文件路径（+ OSC 8 超链接），点一下直接开 L1 peek 到对应行。CLI 输出天然充满路径，这是最便宜的「终端 → 产品」桥。

高成本 · 差异化

多代理赛马

同一个 prompt 同时发给 Pi、codex、claude，各自跑在独立 worktree（ADR-0009 已有多挂载），LiveBoard 并排对比 diff 与测试结果，挑一个合并。外部 CLI 从「客人」变成「赛手」。

低成本

Git 哨兵

外部会话期间 watch git index/工作树，会话结束把累计 diff 汇总成一张变更报告卡（几个文件、多少行、测试是否跑过），作为 Review 入口——即便走的是 D 这种最浅集成。

对比与推荐

方向 live demo 感 实现成本 对 CLI 的假设 风险
A · 终端即房间 强（rail + peek 实时） 中 — PTY + watcher 接入现有房间壳 零（只看文件系统） 时间线无语义；未托管写入需快照兜底
B · 直播舞台 最强 中 — 是 A 的 peek 换个默认姿态 零 跟随跳动，需要节流/钉住
C · 升维卡片流 强且原生 高 — 逐 CLI 翻译层 强（依赖 stream-json / JSONL 格式） 格式升级易碎；TUI 交互无法升维
D · 终端坞 弱（边缘提示） 最低 零 集成感弱，仅作起点
推荐路径：D → A（含 B 作为开关） → C（仅 Claude Code 先行）。
① 先做 D 把 PTY/xterm 基础设施铺好（一周级）；② 把外部会话提升为房间（A），Changes rail 与 L1 peek 由 FS watcher 驱动——这一步几乎全是复用；③ B 不是独立方向，而是 A 里 peek 的 「自动跟随」默认开：ADR-0014 的 live-following 已经存在，舞台=peek 的常开形态；④ C 只对有稳定 结构化输出的 CLI 渐进启用（Claude Code 的 stream-json 最稳），翻译不了的一律回落到 A 的终端形态。 任何方向都先落「安全边界徽章 + 自动快照」，因为外部 CLI 天然在 Tool Gateway 之外。
